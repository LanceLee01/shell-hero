# 2D 肉鸽（Roguelike）小游戏设计方案

## 一、游戏概述

- **类型**：2D 俯视角 Roguelite 动作游戏
- **核心玩法**：玩家控制角色在随机生成的地牢中探索、战斗、收集资源，死亡后失去大部分进度但保留部分永久升级
- **目标平台**：PC (Web / Steam)
- **参考游戏**：*The Binding of Isaac*、*Enter the Gungeon*、*Vampire Survivors*

---

## 二、核心系统

### 1. 地图 & 关卡生成（Procedural Generation）

- **房间拼接**：预置多种房间模板（起始房、战斗房、宝箱房、Boss房、商店房、隐藏房），按规则随机拼接
- **走廊连接**：房间之间由走廊连接，走廊中可能触发随机事件
- **分层结构**：每层一个主题（地牢、森林、冰窟、火山），每层 5~8 个房间，第 5 层为 Boss
- **种子系统**：支持输入种子码复现同一张地图

### 2. 角色系统

- **属性**：生命值、攻击力、移动速度、护甲、暴击率、闪避率
- **升级**：每升一级三选一强化（攻击/防御/特殊）
- **武器**：近战（剑、斧、矛）、远程（弓、法杖、枪）、投掷（飞镖、炸弹）
- **道具**：被动道具（叠加效果）、主动道具（有冷却）、消耗品（血瓶、钥匙、炸弹）

### 3. 战斗系统

- **操作**：WASD 移动，鼠标左键攻击/射击，右键翻滚闪避，Q/E 使用道具，空格翻滚
- **敌人 AI**：
  - 近战型：追踪玩家
  - 远程型：保持距离射击
  - 精英型：特殊技能（冲锋、召唤、AOE）
  - Boss：多阶段、弹幕模式
- **受击反馈**：无敌帧、击退效果、屏幕震动

### 4. 永久成长（Meta Progression）

- **全局货币**：晶石（死亡后保留），用于解锁新角色、武器、被动天赋
- **天赋树**：增加初始属性、解锁新机制（二段跳、额外闪避）
- **角色解锁**：每个角色有独特技能和属性倾向

### 5. 资源 & 经济

| 资源 | 用途 | 是否保留 |
|------|------|----------|
| 金币 | 局内商店购买 | ❌ 死亡丢失 |
| 晶石 | 局外天赋/角色解锁 | ✅ 永久保留 |
| 钥匙 | 开启宝箱/门 | ❌ 死亡丢失 |
| 炸弹 | 炸开隐藏墙/敌人 | ❌ 死亡丢失 |

### 6. 难度曲线

- **动态难度**：随时间或层数递增敌人数值、弹幕密度
- **诅咒机制**：每层可自愿选择诅咒（减速、半血等），换取更高奖励
- **时限奖励**：快速通关额外奖励晶石

---

## 三、美术风格

- **风格**：像素风（16×16 或 32×32 像素）
- **调色板**：每层主题色不同（地牢=暗棕/红，森林=绿/黄，冰窟=蓝/白，火山=橙/黑）
- **特效**：简单粒子系统（火焰、冰霜、爆炸、传送门）
- **UI**：极简 HUD（血条、道具栏、小地图、金币/晶石计数）

---

## 四、技术方案

| 模块 | 方案 |
|------|------|
| 引擎 | Unity 2D 或 Godot 4 / Phaser.js (Web) |
| 地图生成 | BSP + 随机房间放置 + A* 路径连接 |
| 碰撞检测 | Unity Physics 2D / Godot Area2D |
| 数据持久化 | JSON / SQLite (本地存档) |
| 随机数 | 确定性随机（种子 → 可复现） |
| 性能优化 | 对象池（子弹/敌人）、Culling、延迟加载 |

### 推荐开发路线

```
Phase 1 —— 核心原型（2周）
├── 玩家移动 + 基础攻击
├── 一个房间 + 一个敌人
├── 简单碰撞 + 受击
└── 基础 UI（血条）

Phase 2 —— 肉鸽循环（2周）
├── 房间拼接生成
├── 多个房间类型
├── 道具/武器拾取
└── 死亡 → 重开流程

Phase 3 —— 丰富内容（4周）
├── 5+ 敌人类型
├── 2个 Boss
├── 3层主题
├── 商店 + 宝箱
└── 永久升级系统

Phase 4 —— 打磨（2周）
├── 粒子特效
├── 音效 + BGM
├── 画面震动/屏幕反馈
├── 种子系统
└── 设置/存档
```

---

## 五、核心循环（Core Loop）

```
进入关卡 → 清空房间 → 拾取奖励 → 选择路线 →
→ 进入下一房间（战斗/宝箱/商店）→ 清空层数 → Boss战 →
→ 进入下一层 / 死亡 → 晶石结算 → 局外升级
```

---

## 六、关键设计决策

- **Rogue** vs **Roguelite**：选择 Roguelite（保留部分进度），降低挫败感
- **实时** vs **回合**：选择实时动作，手感更爽快
- **像素风**：开发成本低，风格化强，独立游戏友好
- **种子系统**：提升可复现性和社区分享价值

---

## 七、成功指标（MVP 阶段）

- 单局游戏时长：15~30 分钟
- 至少 3 层 × 5 房间 + 1 Boss
- 至少 3 种武器 + 10 种道具
- 至少 5 种敌人 + 2 个 Boss
- 局外天赋树 ≥ 10 个节点
